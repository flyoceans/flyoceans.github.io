删除文件

普通删除，注意这只能删除在工作区的文件，删除完以后要commit一下

git clean命令用来从你的工作目录中删除所有没有tracked过的文件

git clean经常和git reset –hard一起结合使用. 记住reset只影响被track过的文件, 所以需要clean来删除没有track过的文件. 结合使用这两个命令能让你的工作目录完全回到一个指定的的状态

什么叫tracked? 什么叫untracked?

其他操作

看版本库和工作区文件的区别

查看分支：git branch

创建分支：git branch

切换分支：git checkout

创建+切换分支：git checkout -b

合并某分支到当前分支：git merge <name某分支>

删除分支：git branch -d

修复bug时，我们会通过创建新的bug分支进行修复，然后合并，最后删除；

当手头工作没有完成时，先把工作现场git stash一下，然后去修复bug，修复后，再git stash pop，回到工作现场。
